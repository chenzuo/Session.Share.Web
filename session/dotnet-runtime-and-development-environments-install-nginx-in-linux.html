.NET Core SDK 在开发环境中的安装

一、Windows下载安装.NET Core SDK

在Microsoft下载Windows下的.NET Core SDK安装包，下载完成后直接安装即可。安装完成后，以管理员方式打开Windows PowerShell执行以下命令，查看是否正确安装。

#>dotnet --info

以下是返回安装成功后的显示信息 

PS C:\WINDOWS\system32> dotnet --info
                                                .NET Core SDK（反映任何 global.json）:
                                                 Version:   2.2.103
                                                 Commit:    8edbc2570a
                                                
                                                运行时环境:
                                                 OS Name:     Windows
                                                 OS Version:  10.0.17134
                                                 OS Platform: Windo
                                                 RID:         win10-x64
                                                 Base Path:   C:\Program Files\dotnet\sdk\2.2.103\
Windows下集成开发环境的安装

Windows下集成开发工具选择Visual Studio 2019，下载安装包后，直接安装即可。

.NET Core 运行实时在目标操作系统（linux）上的安装

在微软官方dotnet站点根据指引进入安装指引，以次执行以下命令完成安装。完成安装后，关于验证是否安装成功请参考视频。

#>sudo rpm -Uvh https://packages.microsoft.com/config/rhel/7/packages-microsoft-prod.rpm
#>sudo yum update
#>sudo yum install aspnetcore-runtime-2.2
Nginx反向代理服务安装:Nginx源码安装方式

一、GCC安装

用于编译Nginx源码的编译环境

#>yum -y install gcc gcc-c++ autoconf automake make

二、PCRE pcre-devel 安装

PCRE(Perl Compatible Regular Expressions) 是一个Perl库，包括 perl 兼容的正则表达式库。nginx 的 http 模块使用 pcre 来解析正则表达式，所以需要在 linux 上安装 pcre 库，pcre-devel 是使用 pcre 开发的一个二次开发库。nginx也需要此库。 

#>yum install -y pcre pcre-devel

三、zlib库安装

zlib 库提供了很多种压缩和解压缩的方式， nginx 使用 zlib 对 http 包的内容进行 gzip ,如果要在nginx中启用这个功能，必须安装zlib模块。

#>yum install -y zlib zlib-devel

四、OpenSSL 安装

OpenSSL是安全套接字层密码库，囊括主要的密码算法、常用的密钥和证书封装管理功能及 SSL 协议，并提供丰富的应用程序供测试或其它目的使用。 nginx 不仅支持 http 协议，还支持 https（即在ssl协议上传输http）。如果要在nginx中启用此功能，必须在nginx中编译时就应一起编译安装此库。 

#>yum install -y openssl openssl-devel

五、wget方式下载安装包到linux指定目录下

用wget方式直接下载nginx目标安装包。 

#>wget -c https://nginx.org/download/nginx-1.15.9.tar.gz

六、解压安装包

#>tar -zxvf nginx-1.15.9.tar.gz

七、编译设置

nginx在编译前允许对编译后的目标进行设置。如果你不设置其他任何选项，那么默认就可以了。我们这里设置https与gzip压缩方式。

#>cd nginx-1.15.9
#>./configure --prefix=/usr/local/nginx --with-http_gzip_static_module --with-http_stub_status_module --with-http_ssl_module --with-pcre

八、编译安装

执行编译安装命令后，就会自动安装到指定位置。如果编译成功会显示成功提示。如在这个过程中出现编译异常，请在通过网络搜索异常信息来解决。

#>make & make install

九、启动Nginx服务器与测试

#>cd /usr/local/nginx/sbin/
#>./nginx 
#>./nginx -s stop
#>./nginx -s quit
#>./nginx -s reload

#>ps aux|grep nginx
